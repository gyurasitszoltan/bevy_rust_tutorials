Kattints — és a karakter odatalál

Nem WASD, hanem egérkattintós mozgás: ahová a pályán kattintasz, oda a karakter kikeresi az utat a falak között (A*), majd simán végigmegy rajta — ütközés-kezeléssel. Plusz: egy ellenfél üldöz, és kóborló (random walk) szörnyek. Végig Bevy 0.18 ECS-szel, shader nélkül.

1. fejezet

A feladat: kattints, és odatalál

A klikk-mozgás (point-and-click) láncolata mindig ugyanaz, motortól függetlenül:

⊞ A klikk-mozgás folyamata
  kattintás (képernyő-px)
      │  viewport_to_world_2d
      ▼
  világ-koordináta  ──►  cella (osztás a cellamérettel)
      │
      ▼  A*(rács, start-cella, cél-cella)
  útvonal = cellák listája  ──►  világ-koordináta waypointok
      │
      ▼  útkövetés (lépdelünk a következő waypoint felé)
  mozgás  +  ütközés (ne menjünk falba, ne csússzunk át sarkon)
Öt lépés: klikk → világ → cella → A* → útkövetés. Az ütközés a végén „rendet rak”, ha a karakter mégis falhoz érne.

Ugyanezt a gépezetet használja az MI is, csak más a cél-cella forrása:

  • Üldözés (pursue): a cél a játékos aktuális cellája — időnként újratervezve.
  • Kóborlás (random walk): a cél egy véletlen járható cella; ha odaért, új célt sorsol.

A szív tehát a pályakereső (A*), amit egyszer megírunk, és háromféleképp etetünk meg célokkal: kattintás, játékos, véletlen.

▲ A rács honnan jön?
A dual-grid tutorial tilemapjéből: minden cella járható vagy nem (fal). A pályakeresés ezen a logikai rácson dolgozik — a látvány (tile-ok) és a logika (járhatóság) szétválik, ahogy ott is.
2. fejezet

A rács mint gráf

Az algoritmusok gráfon dolgoznak: csúcsok (node) és élek (edge). A rácsunk természetesen az: minden cella egy csúcs, és minden cella össze van kötve a szomszédaival (él). Egy fal-cella vagy nincs is a gráfban, vagy nem vezet bele él.

4 vagy 8 irány?

  • 4 irány (fel/le/bal/jobb): egyszerű, „derékszögű” utak. Minden lépés költsége 1.
  • 8 irány (átlókkal): simább, rövidebb utak, de figyelni kell a sarokvágásra — az átlós lépést csak akkor engedd, ha a két érintett egyenes szomszéd is szabad (különben a karakter „átbújik” a fal sarkán). Az átló költsége √2 ≈ 1.41.
⊞ Szomszédok & a sarokvágás tiltása

8 irányban a karakter „levágná” a fal sarkát — ezt tiltjuk, hogy ne ússzon át tömör blokkokon.

Költségek

Az élnek lehet súlya (cost): a normál talaj 1, a mocsár/homok 3 (lassabb), az út 0.5 (gyorsabb). A pályakereső a legkisebb összköltségű utat keresi — így a karakter kerülni fogja a mocsarat, ha van jobb út. Egységnyi (mindenhol 1) költséggel a „legolcsóbb” = „legkevesebb lépés”.

3. fejezet

BFS, Dijkstra, A*

Három klasszikus algoritmus, egyazon ötlet egyre okosabb változatai. Mind a kiindulóponttól kifelé terjeszkedik, cellákat „felfedezve”, amíg el nem éri a célt. A különbség: milyen sorrendben dolgozzák fel a cellákat.

AlgoritmusSorrend (prioritás)Optimális?Mit fedez fel
BFSérkezési sorrend (FIFO)Igen (egységnyi költség)Mindent egyenletesen, körkörösen
Dijkstralegkisebb eddigi költség gIgen (súlyozott is)Mindent, a költség szerint
Greedybecsült hátralévő hNemKeveset, de a cél felé „nekiront”
A*f = g + hIgen (jó heurisztikával)Keveset, célirányosan

Az A* egyesíti a kettő erényét: a Dijkstra biztos optimalitását (g = eddigi tényleges költség) és a Greedy célirányosságát (h = becsült hátralévő út, a heurisztika). A g + h prioritás miatt csak a cél felé eső cellákat vizsgálja — gyorsabb, mégis optimális.

Nézd meg élőben! Állítsd az algoritmust, és figyeld, hány cellát kell felfedeznie ugyanahhoz az úthoz. Kattints a pályára a cél áthelyezéséhez, jobb gombbal (vagy húzással) rajzolj falat.

Heurisztikák

A heurisztika (h) az A* „iránytűje”: megbecsüli, mennyi van még hátra a célig. A jó heurisztika soha nem becsül felül (admissible) — különben az A* gyors lehet, de nem feltétlenül optimális. A választás a mozgási irányoktól függ:

HeurisztikaKépletMikor
Manhattan|dx| + |dy|4 irányú mozgás — pont a lépésszám
Chebyshevmax(|dx|, |dy|)8 irány, ha az átló is 1 költségű
Octile(dx+dy) + (√2−2)·min(dx,dy)8 irány, ha az átló √2 — a leg­pontosabb
Euklideszi√(dx² + dy²)Szabad (bármilyen szögű) mozgás
⚠ A rossz heurisztika ára
Ha felülbecsülsz (pl. euklideszit használsz 4 irányú mozgásnál, megszorozva), az A* gyorsabb lesz, de nem garantál optimális utat. Ha alulbecsülsz (pl. h = 0), visszakapod a Dijkstrát — optimális, de lassú. A cél a kettő közt: minél pontosabb becslés a felülbecslés határáig.

A fenti vizualizálóban a 8 irány bekapcsolásával az A* automatikusan az octile-szerű, átlós utakat preferálja. Próbáld ki: 8 irányban simább, „rézsútos” utat kapsz.

Útkövetés & ütközés

Az A* cellák listáját adja. Ezeket világ-koordináta waypointokká alakítjuk (a cella közepe), és a karakter mindig a következő waypoint felé lép. Ha elég közel ér, ráugrik és a sorban következőre vált — amíg el nem fogy az út.

Miért kell még külön ütközés?

A pályakeresés a cellák közepei közt vezet, de a karakternek mérete van (egy sugár). Átlós utaknál vagy szűk folyosóban a kör „belelóghat” a sarokba. Ezért a mozgás után egy ütközés-feloldó lépés kitolja a kört a fal-cellákból — ettől a karakter szépen elcsúszik a falak mentén ahelyett, hogy beragadna.

Projekt + a rács erőforrás

A pályakeresés egy rács erőforráson dolgozik: egy járhatósági térkép. Hozzá kell még a világ ↔ cella átváltás. A rendszereket láncba tesszük, hogy a sorrend stabil legyen (előbb cél, aztán mozgás, végül ütközés).

cd clicknav

src/main.rs — rács + átváltás
use bevy::prelude::*;

const CELL: f32 = 16.0;   // egy cella mérete world unitban

#[derive(Resource)]
struct Grid {
    width: i32,
    height: i32,
    walls: Vec<bool>,     // width*height; true = NEM járható
}

impl Grid {
    fn idx(&self, c: IVec2) -> usize { (c.y * self.width + c.x) as usize }
    fn in_bounds(&self, c: IVec2) -> bool {
        c.x >= 0 && c.y >= 0 && c.x < self.width && c.y < self.height
    }
    fn walkable(&self, c: IVec2) -> bool {
        self.in_bounds(c) && !self.walls[self.idx(c)]
    }
}

// világ-koordináta -> cella, és vissza (a cella KÖZEPÉRE)
fn world_to_cell(p: Vec2) -> IVec2 {
    IVec2::new((p.x / CELL).floor() as i32, (p.y / CELL).floor() as i32)
}
fn cell_to_world(c: IVec2) -> Vec2 {
    Vec2::new((c.x as f32 + 0.5) * CELL, (c.y as f32 + 0.5) * CELL)
}
⚙ Bevy fogalom — IVec2 mint cella-kulcs

A cellákat IVec2-vel (egész 2D vektor) címezzük. Ez Copy + Hash + Eq, így közvetlenül használható HashMap/HashSet kulcsként az A*-ban — nem kell külön index-számolgatás a halmazokhoz. A Grid.walls viszont lapos Vec<bool> a gyors indexelésért.

A* Rustban

Itt a lényeg: egy önálló astar függvény. Egy prioritási sor (min-heap) tartja a feldolgozandó cellákat f = g + h szerint; a came_from jegyzi, honnan értünk egy cellába (ebből rakjuk össze az utat); a g_score az eddigi legjobb költség.

src/pathfinding.rs — A*
use bevy::prelude::*;
use std::collections::{BinaryHeap, HashMap};
use std::cmp::Ordering;

const DIRS: [IVec2; 4] = [
    IVec2::new(1, 0), IVec2::new(-1, 0), IVec2::new(0, 1), IVec2::new(0, -1),
];

// a prioritási sor eleme: a cella és az f-értéke
#[derive(Copy, Clone, PartialEq)]
struct Node { f: f32, cell: IVec2 }
impl Eq for Node {}
impl Ord for Node {
    fn cmp(&self, other: &Self) -> Ordering {
        // a BinaryHeap MAX-heap -> megfordítjuk, hogy a KISEBB f jöjjön elöl
        other.f.partial_cmp(&self.f).unwrap_or(Ordering::Equal)
    }
}
impl PartialOrd for Node {
    fn partial_cmp(&self, other: &Self) -> Option<Ordering> { Some(self.cmp(other)) }
}

// Manhattan-távolság — 4 irányú mozgáshoz admissible (sosem becsül felül)
fn heuristic(a: IVec2, b: IVec2) -> f32 {
    ((a.x - b.x).abs() + (a.y - b.y).abs()) as f32
}

/// A legrövidebb út `start`-ból `goal`-ba, vagy None ha nincs.
pub fn astar(grid: &Grid, start: IVec2, goal: IVec2) -> Option<Vec<IVec2>> {
    if !grid.walkable(goal) || !grid.walkable(start) { return None; }

    let mut open = BinaryHeap::new();
    let mut came_from: HashMap<IVec2, IVec2> = HashMap::new();
    let mut g_score: HashMap<IVec2, f32> = HashMap::new();

    g_score.insert(start, 0.0);
    open.push(Node { f: heuristic(start, goal), cell: start });

    while let Some(Node { cell, .. }) = open.pop() {
        if cell == goal {
            return Some(reconstruct(&came_from, goal));
        }
        let g = g_score[&cell];
        for dir in DIRS {
            let next = cell + dir;
            if !grid.walkable(next) { continue; }

            let tentative = g + 1.0;   // minden lépés költsége 1
            if tentative < *g_score.get(&next).unwrap_or(&f32::INFINITY) {
                came_from.insert(next, cell);
                g_score.insert(next, tentative);
                open.push(Node { f: tentative + heuristic(next, goal), cell: next });
            }
        }
    }
    None // a cél nem érhető el
}

// a came_from láncot a célból visszafelé követve rakjuk össze az utat
fn reconstruct(came_from: &HashMap<IVec2, IVec2>, goal: IVec2) -> Vec<IVec2> {
    let mut path = vec![goal];
    let mut cur = goal;
    while let Some(&prev) = came_from.get(&cur) {
        cur = prev;
        path.push(cur);
    }
    path.reverse();   // start -> goal sorrend
    path
}
▲ 8 irány & súlyok
8 irányhoz vedd fel az átlós DIRS-eket (költség std::f32::consts::SQRT_2), tiltsd a sarokvágást (mindkét egyenes szomszéd legyen járható), és cseréld a heurisztikát octile-ra. Súlyozott talajhoz a + 1.0 helyett a belépett cella költségét add (grid.cost(next)) — így lesz Dijkstra/A* mocsárral, úttal.

9. fejezet

Kattintásos mozgás

A kattintást világ-koordinátává alakítjuk (viewport_to_world_2d), abból cellát csinálunk, lefuttatjuk az A*-ot a játékos cellájából, és a kapott utat Path komponensként rátesszük a játékosra.

src/main.rs — click_to_move
fn click_to_move(
    mouse: Res<ButtonInput<MouseButton>>,
    windows: Query<&Window>,
    camera: Query<(&Camera, &GlobalTransform)>,
    grid: Res<Grid>,
    mut commands: Commands,
    player: Query<(Entity, &Transform), With<Player>>,
) {
    if !mouse.just_pressed(MouseButton::Left) { return; }

    let Ok(window) = windows.single() else { return };
    let Some(cursor) = window.cursor_position() else { return };
    let Ok((cam, cam_tf)) = camera.single() else { return };
    let Ok(world) = cam.viewport_to_world_2d(cam_tf, cursor) else { return };

    let Ok((entity, tf)) = player.single() else { return };
    let start = world_to_cell(tf.translation.truncate());
    let goal = world_to_cell(world);

    if let Some(cells) = astar(&grid, start, goal) {
        // cellák -> világ-koordináta waypointok
        let points = cells.iter().map(|c| cell_to_world(*c)).collect();
        commands.entity(entity).insert(Path { points, index: 0 });
    }
}

Útkövetés + ütközés

A follow_path a következő waypoint felé lépteti az entitást; ha elérte, vált a következőre, a végén leveszi a Path-t. A collide_walls utána kitolja a kört a fal-cellákból.

src/main.rs — follow_path
fn follow_path(
    time: Res<Time>,
    mut commands: Commands,
    mut q: Query<(Entity, &mut Transform, &Mover, &mut Path)>,
) {
    for (e, mut tf, mover, mut path) in &mut q {
        if path.index >= path.points.len() {
            commands.entity(e).remove::<Path>();   // odaértünk
            continue;
        }
        let target = path.points[path.index];
        let pos = tf.translation.truncate();
        let to = target - pos;
        let step = mover.speed * time.delta_secs();

        if to.length() <= step {
            tf.translation = target.extend(tf.translation.z); // pattanjunk a waypointra
            path.index += 1;                                   // következő
        } else {
            let dir = to.normalize_or_zero();
            tf.translation += (dir * step).extend(0.0);
        }
    }
}
src/main.rs — collide_walls (kör vs fal-AABB)
fn collide_walls(grid: Res<Grid>, mut q: Query<&mut Transform, With<Mover>>) {
    let radius = CELL * 0.35;
    for mut tf in &mut q {
        let mut p = tf.translation.truncate();
        let cell = world_to_cell(p);

        // a 3x3 szomszédos cellát nézzük végig
        for dy in -1..=1 {
            for dx in -1..=1 {
                let c = cell + IVec2::new(dx, dy);
                if grid.walkable(c) { continue; }   // csak a falak taszítanak

                // a fal-cella AABB-je, és a kör középpontjának legközelebbi pontja rajta
                let min = Vec2::new(c.x as f32 * CELL, c.y as f32 * CELL);
                let max = min + Vec2::splat(CELL);
                let closest = p.clamp(min, max);

                let delta = p - closest;
                let dist = delta.length();
                if dist < radius && dist > 0.0 {
                    p += delta / dist * (radius - dist); // told ki a falból
                }
            }
        }
        tf.translation = p.extend(tf.translation.z);
    }
}
▲ Ágensek széttolása (separation)
Két karakter közt ugyanez az ötlet: ha a köreik átfednek (táv < r1 + r2), told szét őket az átfedés felével mindkét irányba. Sok ágensnél ezt párokra futtatod (vagy térbeli ráccsal gyorsítod), de a logika ugyanaz a „kitolás”.
11. fejezet

Üldözés

Az üldöző MI ugyanazt a Path-gépezetet használja, csak a célja a játékos cellája. Kulcskérdés a gyakoriság: ne tervezz újra minden frame-ben (drága és felesleges), hanem időzítve — pár tizedmásodpercenként. A follow_path a két újratervezés között simán viszi tovább.

src/main.rs — pursue
fn pursue(
    time: Res<Time>,
    grid: Res<Grid>,
    mut commands: Commands,
    player: Query<&Transform, With<Player>>,
    mut pursuers: Query<(Entity, &Transform, &mut Pursuer), Without<Player>>,
) {
    let Ok(player_tf) = player.single() else { return };
    let goal = world_to_cell(player_tf.translation.truncate());

    for (e, tf, mut pursuer) in &mut pursuers {
        pursuer.repath.tick(time.delta());
        if !pursuer.repath.just_finished() { continue; }   // csak időzítve

        let start = world_to_cell(tf.translation.truncate());
        if let Some(cells) = astar(&grid, start, goal) {
            let points = cells.iter().map(|c| cell_to_world(*c)).collect();
            commands.entity(e).insert(Path { points, index: 0 });
        }
    }
}

Kóborlás (random walk)

A kóborló szörny akkor sorsol új célt, amikor nincs útja (épp megérkezett vagy még nem indult). A Without<Path> szűrő elegánsan kifejezi ezt: a rendszer csak az „üres kezű” kóborlókat látja.

src/main.rs — wander
use rand::Rng;

fn wander(
    time: Res<Time>,
    grid: Res<Grid>,
    mut commands: Commands,
    // CSAK az kap új célt, akinek épp NINCS útja:
    mut q: Query<(Entity, &Transform, &mut Wanderer), Without<Path>>,
) {
    let mut rng = rand::thread_rng();
    for (e, tf, mut w) in &mut q {
        w.wait.tick(time.delta());
        if !w.wait.just_finished() { continue; }   // pihen egy kicsit a célok közt

        // sorsolunk egy véletlen JÁRHATÓ cellát
        let mut goal;
        loop {
            goal = IVec2::new(rng.gen_range(0..grid.width), rng.gen_range(0..grid.height));
            if grid.walkable(goal) { break; }
        }

        let start = world_to_cell(tf.translation.truncate());
        if let Some(cells) = astar(&grid, start, goal) {
            let points = cells.iter().map(|c| cell_to_world(*c)).collect();
            commands.entity(e).insert(Path { points, index: 0 });
        }
    }
}
// spawn: Wanderer { wait: Timer::from_seconds(0.8, TimerMode::Repeating) }

▶ JÁTSZÓTÉR — kóborlás

Három szörny kóborol: mind véletlen járható cellát sorsol, A*-ral odamegy, pihen, és új célt vesz.

A halvány vonalak az aktuális útvonalaik. Egymást is kerülgetik (separation).

▲ Olcsóbb kóborlás A* nélkül
Ha a kóborlásnak nem kell a legrövidebb út (csak „menjen valamerre”), elég egy egyszerű steering: lépj egy véletlen irányba, és ha falnak ütköznél, válassz újat. Az A*-os változat akkor jó, ha a szörny tényleg konkrét pontokat látogat (őrjárat, betakarítás).
13. fejezet

A teljes main.rs

Az egész egyben: rács, játékos (kattintós mozgás), egy üldöző, két kóborló, útkövetés és ütközés. A rendszereket láncba tesszük: előbb a célt adók (click_to_move, pursue, wander), majd a mozgás (follow_path), végül az ütközés (collide_walls).

src/main.rs — teljes (az astar a 8. fejezetből)
use bevy::prelude::*;
use rand::Rng;

const CELL: f32 = 16.0;

fn main() {
    App::new()
        .add_plugins(DefaultPlugins.set(ImagePlugin::default_nearest()))
        .insert_resource(make_grid())
        .add_systems(Startup, setup)
        .add_systems(Update, (
            click_to_move, pursue, wander,   // célt adnak (Path-t tesznek)
            follow_path,                     // mozgatnak az úton
            collide_walls,                   // kitolnak a falból
        ).chain())
        .run();
}

// ---------- rács ----------
#[derive(Resource)]
struct Grid { width: i32, height: i32, walls: Vec<bool> }
impl Grid {
    fn idx(&self, c: IVec2) -> usize { (c.y * self.width + c.x) as usize }
    fn in_bounds(&self, c: IVec2) -> bool { c.x >= 0 && c.y >= 0 && c.x < self.width && c.y < self.height }
    fn walkable(&self, c: IVec2) -> bool { self.in_bounds(c) && !self.walls[self.idx(c)] }
}
fn world_to_cell(p: Vec2) -> IVec2 { IVec2::new((p.x / CELL).floor() as i32, (p.y / CELL).floor() as i32) }
fn cell_to_world(c: IVec2) -> Vec2 { Vec2::new((c.x as f32 + 0.5) * CELL, (c.y as f32 + 0.5) * CELL) }

fn make_grid() -> Grid {
    let (w, h) = (30, 20);
    let mut walls = vec![false; (w * h) as usize];
    // pár fal-blokk (a valóságban a tilemapből töltöd)
    for y in 5..15 { walls[(y * w + 10) as usize] = true; }
    for x in 10..20 { walls[(8 * w + x) as usize] = true; }
    Grid { width: w, height: h, walls }
}

// ---------- komponensek ----------
#[derive(Component)] struct Mover { speed: f32 }
#[derive(Component)] struct Path { points: Vec<Vec2>, index: usize }
#[derive(Component)] struct Player;
#[derive(Component)] struct Pursuer { repath: Timer }
#[derive(Component)] struct Wanderer { wait: Timer }

// ---------- setup ----------
fn setup(mut commands: Commands, asset_server: Res<AssetServer>) {
    commands.spawn(Camera2d);

    let dude = asset_server.load("dude.png");
    commands.spawn((Player, Mover { speed: 90.0 },
        Sprite::from_image(dude.clone()), Transform::from_translation(cell_to_world(IVec2::new(2, 2)).extend(10.0))));

    commands.spawn((Pursuer { repath: Timer::from_seconds(0.4, TimerMode::Repeating) },
        Mover { speed: 70.0 }, Sprite::from_image(asset_server.load("enemy.png")),
        Transform::from_translation(cell_to_world(IVec2::new(27, 17)).extend(10.0))));

    for start in [IVec2::new(5, 16), IVec2::new(24, 4)] {
        commands.spawn((Wanderer { wait: Timer::from_seconds(0.8, TimerMode::Repeating) },
            Mover { speed: 55.0 }, Sprite::from_image(asset_server.load("slime.png")),
            Transform::from_translation(cell_to_world(start).extend(10.0))));
    }
}

// ---------- célt adó rendszerek ----------
fn click_to_move(
    mouse: Res<ButtonInput<MouseButton>>, windows: Query<&Window>,
    camera: Query<(&Camera, &GlobalTransform)>, grid: Res<Grid>,
    mut commands: Commands, player: Query<(Entity, &Transform), With<Player>>,
) {
    if !mouse.just_pressed(MouseButton::Left) { return; }
    let Ok(window) = windows.single() else { return };
    let Some(cursor) = window.cursor_position() else { return };
    let Ok((cam, cam_tf)) = camera.single() else { return };
    let Ok(world) = cam.viewport_to_world_2d(cam_tf, cursor) else { return };
    let Ok((entity, tf)) = player.single() else { return };
    if let Some(cells) = astar(&grid, world_to_cell(tf.translation.truncate()), world_to_cell(world)) {
        commands.entity(entity).insert(Path { points: cells.iter().map(|c| cell_to_world(*c)).collect(), index: 0 });
    }
}

fn pursue(
    time: Res<Time>, grid: Res<Grid>, mut commands: Commands,
    player: Query<&Transform, With<Player>>,
    mut pursuers: Query<(Entity, &Transform, &mut Pursuer), Without<Player>>,
) {
    let Ok(player_tf) = player.single() else { return };
    let goal = world_to_cell(player_tf.translation.truncate());
    for (e, tf, mut p) in &mut pursuers {
        p.repath.tick(time.delta());
        if !p.repath.just_finished() { continue; }
        if let Some(cells) = astar(&grid, world_to_cell(tf.translation.truncate()), goal) {
            commands.entity(e).insert(Path { points: cells.iter().map(|c| cell_to_world(*c)).collect(), index: 0 });
        }
    }
}

fn wander(
    time: Res<Time>, grid: Res<Grid>, mut commands: Commands,
    mut q: Query<(Entity, &Transform, &mut Wanderer), Without<Path>>,
) {
    let mut rng = rand::thread_rng();
    for (e, tf, mut w) in &mut q {
        w.wait.tick(time.delta());
        if !w.wait.just_finished() { continue; }
        let mut goal = IVec2::ZERO;
        loop { goal = IVec2::new(rng.gen_range(0..grid.width), rng.gen_range(0..grid.height));
               if grid.walkable(goal) { break; } }
        if let Some(cells) = astar(&grid, world_to_cell(tf.translation.truncate()), goal) {
            commands.entity(e).insert(Path { points: cells.iter().map(|c| cell_to_world(*c)).collect(), index: 0 });
        }
    }
}

// ---------- mozgás + ütközés ----------
fn follow_path(time: Res<Time>, mut commands: Commands, mut q: Query<(Entity, &mut Transform, &Mover, &mut Path)>) {
    for (e, mut tf, mover, mut path) in &mut q {
        if path.index >= path.points.len() { commands.entity(e).remove::<Path>(); continue; }
        let target = path.points[path.index];
        let to = target - tf.translation.truncate();
        let step = mover.speed * time.delta_secs();
        if to.length() <= step { tf.translation = target.extend(tf.translation.z); path.index += 1; }
        else { tf.translation += (to.normalize_or_zero() * step).extend(0.0); }
    }
}

fn collide_walls(grid: Res<Grid>, mut q: Query<&mut Transform, With<Mover>>) {
    let radius = CELL * 0.35;
    for mut tf in &mut q {
        let mut p = tf.translation.truncate();
        let cell = world_to_cell(p);
        for dy in -1..=1 { for dx in -1..=1 {
            let c = cell + IVec2::new(dx, dy);
            if grid.walkable(c) { continue; }
            let min = Vec2::new(c.x as f32 * CELL, c.y as f32 * CELL);
            let closest = p.clamp(min, min + Vec2::splat(CELL));
            let delta = p - closest;
            let dist = delta.length();
            if dist < radius && dist > 0.0 { p += delta / dist * (radius - dist); }
        }}
        tf.translation = p.extend(tf.translation.z);
    }
}

Merre tovább?

  • Kész crate. A pathfinding crate astar/bfs/dijkstra-ja kipróbált és gyors — éles projektben ezt hívd.
  • Sok ágens, egy cél: flow field. Ha 100 szörny ugyanazt a játékost üldözi, ne fuss 100 A*-ot. Számolj egyszer egy „áramlási mezőt” (minden cella egy nyilat tárol a cél felé), és minden ágens csak a nyilat követi.
  • Nagy pálya: hierarchikus pathfinding. Oszd régiókra (HPA*), tervezz előbb régiók közt, aztán cellaszinten — nagyságrendekkel gyorsabb.
  • Navmesh. Nyílt, nem rács-alapú világhoz az oxidized_navigation (Bevy) navmesh-t ad — sokszögekre, nem cellákra tervez.
  • Steering & útsimítás. Az A* út szögletes; egy „string-pulling”/line-of-sight simítás levágja a felesleges töréseket, a steering-viselkedések (seek, avoid) emberibb mozgást adnak.
  • Újratervezés okosan. Üldözésnél csak akkor tervezz újra, ha a játékos cellát váltott (vagy elromlott az út) — nem vakon időzítve.

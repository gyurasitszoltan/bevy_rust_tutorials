String, &str és az iterátor-eszköztár

A két típus, ami minden kezdőt elejt, és az absztrakció, amit naponta használsz. Itt a gyakorlat felől nézzük meg őket — és a végén összeér a kettő: szövegfeldolgozás iterátorláncokkal.

10 fejezet ~40 kódpélda Szint: középhaladó Épít rá: a nagy Rust-tananyag (Vec, String, closure-ök)
0x00

A két szövegtípus

A Rust egyik leggyakoribb kezdő-buktatója, hogy kétféle szöveg van. Nem azért, hogy bosszantson — a kettő két különböző dolgot jelent, és ha érted a különbséget, a szöveggel kapcsolatos fordítási hibák fele megszűnik.

TípusMi ezTulajdonos?
StringSaját, a heap-en tárolt, növelhető szöveg.Igen — ő birtokolja az adatot.
&strKölcsönzött, fix nézet egy meglévő szövegre („string slice").Nem — csak rámutat.
src/main.rs
fn main() {
    // &str — kölcsönzött, fix nézet. A string literál típusa &'static str:
    let kolcson: &str = "Helló";

    // String — saját tulajdonú, a heap-en, NÖVELHETŐ:
    let mut sajat: String = String::from("Helló");
    sajat.push_str(", világ!"); // egy &str-t nem tudnál bővíteni

    println!("{kolcson} | {sajat}");
}
A motorháztető alatt

A String három dolgot tárol a stacken: egy mutatót a heap-adatra, a hosszt és a kapacitást. A &str csak kettőt: egy mutatót és a hosszt — ezért „kövér mutató" (fat pointer). Egy &str rámutathat egy String adatára, egy literálra (a program .rodata szegmensében), vagy bármilyen UTF-8 bájtsorozatra. Nem másolat — nézet.

0x01

Melyiket az argumentumban?

Ez a fejezet egyetlen szabályról szól, amit ha megjegyzel, profibb API-kat írsz:

Az aranyszabály

Kérj &str-t a paraméterben, és adj vissza String-et. A &str a legrugalmasabb bemenet: literállal és String-gel is hívható. A String visszaadásával pedig a hívó megkapja a tulajdont, és azt csinál vele, amit akar.

src/main.rs
// JÓ: &str-t kérünk → hívható literállal IS, String-gel IS.
fn koszon(nev: &str) {
    println!("Szia, {nev}!");
}

fn main() {
    let tulajdon = String::from("Anna");

    koszon("Béla");     // &str literál — közvetlenül megy
    koszon(&tulajdon);  // &String → &str AUTOMATIKUSAN (deref coercion)
}

A &tulajdon egy &String, de a függvény &str-t vár — a fordító ezt magától elintézi. Ezt hívják deref coercion-nek, és ezért nincs soha értelme &String-et kérni a paraméterben.

Gyakori kezdő-hiba

Ne kérj fn koszon(nev: String)-t, ha csak olvasod! Azzal elveszed a tulajdont a hívótól (move), aki utána nem használhatja a változóját — és literállal is csak egy felesleges .to_string()-gel hívható. A &str mindkét gondot megoldja.

0x02

UTF-8 — amit tudni kell

A Rust minden szövege UTF-8 kódolású. Ez nagyszerű (a világ minden nyelve működik), de három dolgot meg kell érteni, különben futásidejű pánikba szaladsz.

src/main.rs
fn main() {
    let s = "héló"; // h · é · l · ó

    // 1) A .len() BÁJTOKAT ad, nem karaktereket!
    //    Az é és az ó 2-2 bájt UTF-8-ban, a h és az l 1-1.
    println!("{}", s.len());            // 6  (NEM 4!)
    println!("{}", s.chars().count());  // 4  — ennyi a karakter

    // 2) NINCS s[0] indexelés — ez fordítási hiba.
    // let c = s[0];                     // ❌ a bájt-index félbevághatna egy karaktert

    // 3) Karakterek és pozícióik bejárása:
    for (bajt_index, c) in s.char_indices() {
        println!("{bajt_index}: {c}");  // 0:h  1:é  3:l  4:ó
    }
}

Szeletelés — itt csússz el

Szeletelni lehet, de a bájt-indexnek karakterhatárra kell esnie, különben futásidejű pánik. Ha bizonytalan az index, használd a pánik helyett Option-t adó .get()-et.

src/main.rs
fn main() {
    let s = "héló";

    let eleje = &s[0..1];        // "h" — a 0..1 bájthatáron van, rendben
    println!("{eleje}");

    // let rossz = &s[0..2];     // ❌ PÁNIK: byte index 2 is not a char boundary (az é közepe)

    // Biztonságos változat: get() — None-t ad pánik helyett
    println!("{:?}", s.get(0..1)); // Some("h")
    println!("{:?}", s.get(0..2)); // None
}
Karakter, bájt, vagy ennél is bonyolultabb?

A .chars() Unicode kódpontokat ad. A legtöbb feladatra ez kell. Ami a felhasználó szemében „egy betű" (pl. emoji bőrszín-módosítóval, vagy összetett ékezet), az lehet több kódpont is — ezt „grapheme cluster"-nek hívják, és a unicode-segmentation crate kezeli. A mindennapokban a .chars() bőven elég; csak tudd, hogy létezik a réteg alatta.

0x03

Építés és metódus-katalógus

String építése

src/main.rs
fn main() {
    let mut s = String::new();
    s.push_str("Helló");   // &str hozzáfűzése
    s.push(' ');           // EGYETLEN karakter (char) hozzáfűzése
    s.push_str("világ");

    // + operátor: a BAL oldalt elfogyasztja (move), a jobb oldal &str:
    let teljes = s + "!";  // s elmozgott; teljes == "Helló világ!"

    // format! — semmit nem mozgat, és messze a legolvashatóbb építő:
    let nev = "Anna";
    let kor = 30;
    let mondat = format!("{nev} {kor} éves.");

    println!("{teljes}");
    println!("{mondat}");
}
Egyszerű szabály

Kettőnél több darab összefűzéséhez használj format!-ot — olvashatóbb, mint a + láncolgatása, és nem kell a move-okkal bajlódni. Ciklusban épülő szöveghez a push_str a leghatékonyabb.

src/main.rs
fn main() {
    let s = "  alma,körte,szilva  ";

    println!("{:?}", s.trim());                 // "alma,körte,szilva"

    let darabok: Vec<&str> = s.trim().split(',').collect();
    println!("{darabok:?}");                    // ["alma", "körte", "szilva"]

    println!("{}", "Helló".to_uppercase());     // "HELLÓ"
    println!("{}", "alma".contains("lm"));      // true
    println!("{}", "Cargo.toml".ends_with(".toml")); // true
}
⚙ Gyakorlat 0x03

Írj fn maganhangzok(s: &str) -> usize függvényt, amely megszámolja a magánhangzókat egy (kisbetűs) szövegben. Tipp: .chars() + .filter(..) + .count(), és a &str-nek van .contains(char) metódusa.

Megoldás
megoldás
fn maganhangzok(s: &str) -> usize {
    s.chars()
        .filter(|c| "aeiouáéíóöőúüű".contains(*c))
        .count()
}
0x04

Lustaság: adapter vs consumer

Itt kezdődik a második rész — és a Rust egyik legkedveltebb eszköze. Az iterátorok megértésének kulcsa egyetlen szó: lustaság. Az iterátor-műveletek két csoportra oszlanak:

  • Adapter (pl. map, filter): új iterátort ad vissza. Semmit nem csinál, csak „receptet" épít.
  • Consumer (pl. collect, sum, count): elindítja a láncot, és előállítja a végeredményt.
src/main.rs
fn main() {
    let szamok = vec![1, 2, 3, 4, 5];

    // EZ A SOR SEMMIT NEM SZÁMOL — az adapterek lusták (lazy):
    let lanc = szamok.iter().map(|&x| x * 2).filter(|&x| x > 4);
    //         ^ egyetlen szorzás vagy szűrés sem futott le; `lanc` csak egy recept.

    // A consumer indítja el az EGÉSZ láncot, egyetlen menetben:
    let eredmeny: Vec<i32> = lanc.collect();
    println!("{eredmeny:?}"); // [6, 8, 10]
}
Miért jó ez?

Mert a lánc egyetlen menetben fut le, elemenként — nem készül köztes Vec minden lépés után. A map majd filter nem két bejárás: minden elem végigmegy a teljes láncon, aztán jön a következő. Ezért „zero-cost" az absztrakció: a fordított kód olyan gyors, mint egy kézzel írt for-ciklus. Ha pedig elfelejted a consumert, a fordító figyelmeztet: „unused Map that must be used".

0x05

iter / iter_mut / into_iter

Háromféleképpen kérhetsz iterátort egy gyűjteményből, és a különbség tulajdonjogi: mit kapsz az elemekből — kölcsönt, módosítható kölcsönt, vagy a tulajdont.

HívásElem típusaA gyűjtemény utána
.iter()&T (olvasás)megmarad
.iter_mut()&mut T (módosítás)megmarad, módosulhat
.into_iter()T (tulajdon)elfogy
src/main.rs
fn main() {
    // iter() → &T: csak olvasunk, a vektor utána is megvan
    let v = vec![10, 20, 30];
    for x in v.iter() {
        print!("{x} ");          // x: &i32
    }
    println!("| megvan: {v:?}");

    // iter_mut() → &mut T: helyben módosítunk
    let mut v2 = vec![1, 2, 3];
    for x in v2.iter_mut() {
        *x *= 10;                // x: &mut i32 — a * dereferál
    }
    println!("{v2:?}");          // [10, 20, 30]

    // into_iter() → T: elfogyasztja a vektort
    let v3 = vec![String::from("a"), String::from("b")];
    for s in v3.into_iter() {
        println!("{s}");         // s: String (tulajdon)
    }
    // v3 itt már NEM használható — elfogyott
}
A for-ciklus titka

A for a háttérben ezeket hívja: for x in &v ⇒ v.iter(), for x in &mut v ⇒ v.iter_mut(), és a sima for x in v ⇒ v.into_iter() (vagyis elfogyasztja a gyűjteményt!). Ha a ciklus után még kell a vektor, írj &v-t.

0x06

Transzformáló adapterek

Ezek építik a láncot. Itt a leghasznosabbak, két futtatható csokorban.

map, filter, filter_map, flat_map

src/main.rs
fn main() {
    // filter_map: szűr ÉS átalakít egy lépésben — a None-okat eldobja.
    let nyers = vec!["1", "kettő", "3", "négy", "5"];
    let szamok: Vec<i32> = nyers.iter()
        .filter_map(|s| s.parse().ok())  // a parse cél-típusát a collect adja: i32
        .collect();
    println!("{szamok:?}"); // [1, 3, 5]

    // flat_map: minden elemből TÖBB lesz, majd egy szintbe lapít.
    let mondatok = vec!["a b", "c d e"];
    let szavak: Vec<&str> = mondatok.iter()
        .flat_map(|m| m.split(' '))
        .collect();
    println!("{szavak:?}"); // ["a", "b", "c", "d", "e"]
}

enumerate, zip, take, skip, rev

src/main.rs
fn main() {
    let v = vec![1, 2, 3, 4, 5, 6];

    // enumerate: (index, elem) párokat ad
    for (i, x) in v.iter().enumerate() {
        if i % 2 == 0 { print!("[{i}]={x} "); }
    }
    println!();

    // zip: két iterátort párokká fűz; a rövidebbnél megáll
    let nevek = vec!["Anna", "Béla"];
    let korok = vec![30, 25];
    for (nev, kor) in nevek.iter().zip(korok.iter()) {
        println!("{nev}: {kor}");
    }

    // take / skip / rev — a lánc közben
    let elso_harom: Vec<i32> = v.iter().copied().take(3).collect();   // [1, 2, 3]
    let forditva:   Vec<i32> = v.iter().copied().rev().collect();      // [6, 5, 4, 3, 2, 1]
    println!("{elso_harom:?} | {forditva:?}");
}
A closure-ök referenciái

Az .iter() referenciákat ad, ezért látsz |&x| mintát: a & a mintában „kicsomagolja" a referenciát, így a törzsben x már a sima érték. A .filter() ráadásul referenciát ad a closure-nek, ezért ott néha dupla a jel: |&&x|. Ha belezavarodsz, írhatsz egy *-ot a törzsben helyette (pl. |x| *x > 4).

0x07

Lezáró műveletek és collect

A consumerek zárják le a láncot. Itt a leggyakoribbak.

src/main.rs
fn main() {
    let v = vec![1, 2, 3, 4, 5];

    let osszeg: i32 = v.iter().sum();                       // 15
    let paros_db = v.iter().filter(|&&x| x % 2 == 0).count(); // 2
    let szorzat  = v.iter().fold(1, |acc, &x| acc * x);      // 120 (saját akkumulátor)

    println!("összeg={osszeg}, páros={paros_db}, szorzat={szorzat}");

    // keresők — mind Option-t vagy bool-t adnak:
    println!("{:?}", v.iter().find(|&&x| x > 3));   // Some(4)
    println!("{:?}", v.iter().position(|&x| x == 3)); // Some(2)
    println!("{}",   v.iter().any(|&x| x < 0));       // false
    println!("{}",   v.iter().all(|&x| x > 0));       // true

    // max_by_key: a leghosszabb szó
    let szavak = vec!["alma", "körte", "szilva"];
    println!("{:?}", szavak.iter().max_by_key(|s| s.len())); // Some("szilva")
}

collect — a svájcibicska

A collect abba gyűjt, amit a céltípus megkövetel. A típust a változó annotációja vagy egy „turbofish" (::<...>) adja meg.

src/main.rs
use std::collections::HashMap;

fn main() {
    let v: Vec<i32> = (1..=5).collect();                 // [1, 2, 3, 4, 5]
    let s: String = vec!['R', 'u', 's', 't'].into_iter().collect(); // "Rust"
    let map: HashMap<&str, i32> =
        vec![("a", 1), ("b", 2)].into_iter().collect();  // {"a":1, "b":2}

    println!("{v:?} | {s} | {map:?}");
}
A legfontosabb collect-trükk: Result-ök összegyűjtése

Ha Result-okból álló iterátorod van, a collect képes egyetlen Result<Vec<_>, _>-ba gyűjteni: ha minden elem Ok, megkapod az Ok(Vec)-et; ha bármelyik Err, az első hiba „rövidre zárja" az egészet. Ez a hibatűrő adatfeldolgozás motorja.

src/main.rs
fn main() {
    let mind: Result<Vec<i32>, _> =
        vec!["1", "2", "3"].into_iter().map(|s| s.parse::<i32>()).collect();
    println!("{mind:?}"); // Ok([1, 2, 3])

    let hibas: Result<Vec<i32>, _> =
        vec!["1", "x", "3"].into_iter().map(|s| s.parse::<i32>()).collect();
    println!("{hibas:?}"); // Err(ParseIntError { .. })
}
⚙ Gyakorlat 0x07

Írj fn paros_negyzetosszeg(v: &[i32]) -> i32 függvényt, amely egyetlen iterátorláncban visszaadja a páros számok négyzeteinek összegét.

Megoldás
megoldás
fn paros_negyzetosszeg(v: &[i32]) -> i32 {
    v.iter()
        .filter(|&&x| x % 2 == 0)
        .map(|&x| x * x)
        .sum()
}
0x08

Szövegfeldolgozás a gyakorlatban

Most ér össze a két téma. A való életben a szövegfeldolgozás szinte mindig egy iterátorlánc: sorokra bontasz, szűrsz, átalakítasz, parse-olsz, összegzel.

Sorok szűrése és összegzése egyetlen láncban

src/main.rs
fn main() {
    let szoveg = "alma 3\n# komment\nkörte 5\n\nszilva 2";

    // Üres és kommentsorok kiszűrése, a 2. mező kivétele, parse, összegzés:
    let osszeg: i32 = szoveg
        .lines()
        .map(|sor| sor.trim())
        .filter(|sor| !sor.is_empty() && !sor.starts_with('#'))
        .filter_map(|sor| sor.split_whitespace().nth(1)) // a második mező
        .filter_map(|mezo| mezo.parse::<i32>().ok())
        .sum();

    println!("összeg: {osszeg}"); // 10
}

Szószámláló HashMap-pel

src/main.rs
use std::collections::HashMap;

fn szavak_szamlalasa(szoveg: &str) -> HashMap<String, usize> {
    let mut szamlalo: HashMap<String, usize> = HashMap::new();
    for szo in szoveg.split_whitespace() {
        // entry(...).or_insert(0): ha nincs még a kulcs, beteszi 0-val, majd növel
        *szamlalo.entry(szo.to_lowercase()).or_insert(0) += 1;
    }
    szamlalo
}

fn main() {
    let szamlalo = szavak_szamlalasa("a kutya és a macska és a kutya");
    println!("{szamlalo:?}"); // {"a": 3, "kutya": 2, "és": 2, "macska": 1}
}

Bónusz: windows és chunks

Ezek nem iterátor-adapterek, hanem slice-metódusok — de iterátort adnak, így beilleszthetők a láncba. Szomszédos elemek vizsgálatához aranyat érnek.

src/main.rs
fn main() {
    let v = [1, 2, 3, 4, 5];

    // windows(2): ÁTFEDŐ ablakok — szomszédos párok
    for w in v.windows(2) {
        print!("{w:?} ");   // [1, 2] [2, 3] [3, 4] [4, 5]
    }
    println!();

    // chunks(2): NEM átfedő darabok (az utolsó lehet rövidebb)
    for c in v.chunks(2) {
        print!("{c:?} ");   // [1, 2] [3, 4] [5]
    }
    println!();
}
⚙ Gyakorlat 0x08

Adott egy többsoros szöveg, soronként "név,pont" formátumban (pl. "Anna,90\nBéla,75"). Írj fn atlag(szoveg: &str) -> f64 függvényt, amely egy iterátorlánccal kiszámolja a pontok átlagát (üres lista esetén 0.0).

Megoldás
megoldás
fn atlag(szoveg: &str) -> f64 {
    let pontok: Vec<f64> = szoveg
        .lines()
        .filter_map(|sor| sor.split(',').nth(1))      // a pont-mező
        .filter_map(|p| p.trim().parse::<f64>().ok())  // számmá, hibásat eldobva
        .collect();

    if pontok.is_empty() {
        return 0.0;
    }
    pontok.iter().sum::<f64>() / pontok.len() as f64
}

A leg­fontosabb gondolat

Az iterátorlánc nem lassabb, mint a kézzel írt for-ciklus — a fordító egyenértékű gépi kódot készít belőle (zero-cost absztrakció). Tehát nyugodtan írj olvasható, deklaratív láncokat: a „mit", nem a „hogyan". A teljesítményt nem áldozod fel az érthetőségért.

Merre tovább?

Ez a második hiánypótló modul a magyar Rust-tananyag mellé. A sorozat következő, tervezett darabja:

  • „A fordító a barátod" — a borrow checker hibaüzeneteinek olvasása (E0382, E0499, E0502…), és a hibakeresési munkafolyamat (dbg!, tracing, debugger).

A korábbi modulok: a nagy Rust-tananyag, az Option<T> mélymerülés, és a mintaillesztés + konverziók. És ne feledd: a magabiztosság a gyakorlásból jön — szedd szét ezeket a láncokat, írd át for-ciklussá, aztán vissza. 🦀

Rust → Bevy fordítókártya

A trilógiában megtanult Rust-fogalmak nem tűnnek el Bevyben — konkrét alakot öltenek. Ez a kártya egy lapon megmutatja, melyik fogalom hol jön elő, és miért.

A nagy megfeleltetés

A Bevy ECS-e nem „új nyelv" — ugyanaz a Rust, csak az ownership-modell köré épített API. Ez a táblázat a teljes kártya; a következő fejezetek a fontosabb sorokat bontják ki kis példákkal.

0x01

Tulajdonjog → Query, Commands

Ez a legfontosabb híd. A Query megosztott (&) vagy módosító (&mut) kölcsönt ad a komponensekre — pontosan a Rust kölcsönzési szabályai szerint, csak az ütemező tartatja be (lásd a futásidejű hibák modult).

Miért van Commands? — az E0502 ECS-ben

A Rustban nem mutálhatsz egy gyűjteményt, amíg iterálod (E0502). Ugyanígy nem spawnolhatsz/despawnolhatsz a World-ben query-iterálás közben — ezért van a Commands, ami a strukturális változást a frame végére halasztja.

src/main.rs
fn halal_rendszer(mut commands: Commands, q: Query<(Entity, &Eletero)>) {
    for (entitas, eletero) in &q {        // iterátor: &q == q.iter()
        if eletero.0 <= 0 {
            commands.entity(entitas).despawn(); // halasztott — iterálás közben is biztonságos
        }
    }
}

A Handle az Rc megfelelője

src/main.rs
// A Handle<T> olyan, mint az Rc<T>: olcsó klón, megosztott tulajdon az asseten.
let kep: Handle<Image> = asset_server.load("jatekos.png");
let ugyanaz = kep.clone(); // csak a referenciaszámlálót növeli, NEM másolja a képet
0x02

Trait & derive → Component

A Component, Resource, Event és Message mind trait. Magadnak ritkán implementálod — a derive legenerálja, pont mint a Debug-ot a Rust-oldalon.

src/main.rs
#[derive(Component)]            // entitásra rakható adat
struct Eletero(i32);

#[derive(Resource)]            // globális, egyetlen példányú állapot
struct Pontszam(u32);

#[derive(Message)]             // pufferelt esemény (MessageReader/MessageWriter)
struct Sebzes { cel: Entity, mennyi: i32 }

#[derive(Event)]               // azonnali, observerrel elkapható esemény
struct Robbanas;
Ha kimarad a derive → E0277

Ha elfelejted a #[derive(Component)]-et, és spawnolni vagy query-zni próbálsz egy típust, pont a 3. tutorialból ismert hibát kapod: „the trait bound Eletero: Component is not satisfied" (E0277). A javítás: tedd vissza a derive-ot.

0x03

Generikus, enum, Option, Result

Generikus + trait-bound → a Query szűrői

A Query<adat, szűrő> tele van generikusokkal. A With/Without ugyanaz a szűrő-logika, amit a B0001 javításánál láttál.

src/main.rs
fn rendszer(q: Query<&Transform, (With<Jatekos>, Without<Ellenseg>)>) {
    // csak a játékos-entitások Transform-jai, az ellenségeké nem
}

Enum + match → States (állapotgép)

src/main.rs
#[derive(States, Default, Debug, Clone, PartialEq, Eq, Hash)]
enum Jatekallapot {
    #[default] Menu,
    Jatek,
    Szunet,
}
// rendszer csak egy állapotban: .add_systems(Update, .run_if(in_state(Jatekallapot::Jatek)))

Option és Result a query-kben

src/main.rs
// Option<&T>: "lehet, hogy van rajta a komponens, lehet, hogy nincs"
fn mozgas(q: Query<(&Transform, Option<&Sebesseg>)>) {
    for (hely, sebesseg) in &q {
        match sebesseg {
            Some(s) => { /* mozgó entitás */ }
            None    => { /* álló entitás */ }
        }
    }
}

// single() Result-ot ad (0.18) — let-else, ahogy a String+iterátor modulban:
fn kamera(jatekos: Query<&Transform, With<Jatekos>>) {
    let Ok(hely) = jatekos.single() else { return; };
    // ... a kamera követi a `hely`-et
}
0x04

Closure, iterátor, élettartam

A rendszer egy függvény, az iteráció iterátor

src/main.rs
// A rendszert a PARAMÉTEREI teszik rendszerré (mindegyik egy SystemParam).
fn mozgas(time: Res<Time>, mut q: Query<&mut Transform, With<Jatekos>>) {
    for mut hely in &mut q {              // &mut q == q.iter_mut()
        hely.translation.x += 100.0 * time.delta_secs();
    }
}

From/Into a Bevyben is

src/main.rs
// Sok konstruktor ugyanaz a From/Into minta, amit a konverziók modulban tanultál.
let piros = Color::srgb(1.0, 0.0, 0.0);
let pozicio = Vec3::new(10.0, 0.0, 0.0);
commands.spawn((Sprite::from_color(piros, Vec2::splat(32.0)), Transform::from_translation(pozicio)));
Az élettartamok jó híre

A Query valódi típusa Query<'w, 's, D, F> — tele élettartamokkal ('w = world, 's = state). De ezeket a Bevy elrejti: a rendszereidben sosem írod ki őket, az elízió elintézi. Vagyis a trilógia élettartam-fejezete itt főleg arra kell, hogy értsd, mi történik — nem hogy gépeld.

0x05

Záró és a teljes csomag

Ez a kártya az átkötő a nyelv és a motor közt. A leglényegét egy mondatban: a Bevy nem rejti el a Rustot — láthatóvá teszi. Minden rendszer-paraméter, minden & és &mut a query-ben a tulajdonjogi modell egyenes következménye.

A kiegészítő csomag — öt darab

Rust-oldal: mintaillesztés + konverziók · String + iterátorok · fordítóhibák + debug. Bevy-híd: futásidejű hibák + dev-ciklus · ez a fordítókártya. A nagy kurzusok: Rust-tananyag és Bevy-műhely.

Merre tovább?

Sehová — vagyis a kódszerkesztőbe. Innen már nincs hiányzó réteg a magabiztos játékfejlesztéshez: a nyelv a kezedben, a motor a kezedben, a hibák olvashatók. Nyiss egy cargo new-t, és kezdj el építeni. 🦀
